# Code of Conduct

## Our pledge

We want `gruff-go` to be a useful, professional project for maintainers, contributors, and users. Everyone participating in this project is expected to act with respect, patience, and good faith.

## Expected behavior

- Use welcoming and inclusive language.
- Keep technical criticism specific to the work.
- Assume good intent, but respect clear boundaries when someone asks for a change in tone or behavior.
- Accept responsibility, apologize when appropriate, and correct mistakes promptly.
- Prioritize the community's health and the project's long-term maintainability.

## Unacceptable behavior

The following behavior is not welcome:

- Harassment, intimidation, insults, or sustained disruption.
- Discriminatory language or conduct.
- Threats, stalking, unwanted attention, or sharing private information without consent.
- Spam, impersonation, or deliberately misleading reports.
- Any other conduct a maintainer reasonably determines to be inappropriate in a professional open source setting.

## Scope

This code applies in project spaces, including issues, pull requests, discussions, commits, release assets, and any other project-controlled communication channel. It also applies when someone represents the project in a public space.

## Enforcement

Report concerns to hello@blundergoat.com. Maintainers will review reports privately, avoid conflicts of interest where possible, and respond in a way that is proportionate to the impact and evidence.

Maintainers may remove comments, close issues or pull requests, request behavior changes, temporarily limit participation, or permanently block participation when needed to protect the project.
